# OPSC6311 Part 2 Submission Checklist

Use this checklist before you submit **Part 2 (App Prototype Development)**.

## 1) Core App Features (Prototype Scope)

- [ ] App runs in emulator with no major crashes.
- [ ] Login with username and password works.
- [ ] User can create categories for expenses/budget entries.
- [ ] User can create expense entries with:
  - [ ] date
  - [ ] start time
  - [ ] end time
  - [ ] description
  - [ ] category
- [ ] User can optionally attach a photo to an expense.
- [ ] User can set monthly goals:
  - [ ] minimum spend goal
  - [ ] maximum spend goal
- [ ] User can view a list of entries for a user-selectable period.
- [ ] If an entry has a photo, it can be opened from the list.
- [ ] User can view category totals for a user-selectable period.
- [ ] All data is saved locally (RoomDB/SQLite/offline DB).

## 2) Quality and Stability

- [ ] Invalid input is handled without crashing.
- [ ] UI is user-friendly, consistent, and logical.
- [ ] Only minor bugs remain (if any).
- [ ] App flow is easy for marker to demonstrate quickly.

## 3) GitHub Requirements

- [ ] Project is on GitHub (no zip submission).
- [ ] Repository includes a `README.md`.
- [ ] Commits show regular progress (not one giant commit).
- [ ] Latest code is pushed to remote branch/repo.

## 4) Testing + CI

- [ ] Unit tests for main functionality are included.
- [ ] GitHub Actions workflow is set up.
- [ ] CI runs tests successfully.
- [ ] CI can build app successfully.

## 5) Code Standards Expected

- [ ] Code has meaningful comments.
- [ ] Logging statements are present to show understanding.
- [ ] Project structure is clean and understandable.

## 6) Video Deliverable

- [ ] Professional demo video created.
- [ ] Voice-over explains what is being shown.
- [ ] Video shows all required Part 2 features.
- [ ] Video compressed for easier upload.
- [ ] Video uploaded (e.g., unlisted YouTube).
- [ ] Video link added to `README.md`.

## 7) Final Submission Pack (Part 2)

- [ ] GitHub repo link (with full Kotlin source code).
- [ ] `README.md` in repo (with video link).
- [ ] Built APK file.
- [ ] Demo video accessible to lecturer.

## Quick Self-Check (5-minute pass)

- [ ] Can I install and run the APK quickly?
- [ ] Can I demo all required features in under 5-7 minutes?
- [ ] Does the app avoid crashing during normal use?
- [ ] Does README clearly explain setup + demo link?
- [ ] Is the latest working code pushed to GitHub?
